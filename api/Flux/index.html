Flux

Client module for mounting and managing GUI instances. Each mount parents the GUI to PlayerGui and returns a Controller that owns the instance's lifecycle. The caller is responsible for cloning templates before passing them in. 

local Flux = require(ReplicatedStorage.Flux)

-- Create reactive state
local coins: Flux.State<number> = Flux:State(0)

-- Clone and mount the GUI
local gui = ReplicatedStorage.Assets.Gui.CoinScreenGui:Clone()
local controller = Flux:MountScreenGui(gui)

-- Bind state to UI
controller:Bind(coins, function(count: number)
	local frame = controller:GetInstance():FindFirstChild("Frame")
	if frame then
		local coinsLabel = frame:FindFirstChild("CoinsLabel")
		if coinsLabel and coinsLabel:IsA("TextLabel") then
			coinsLabel.Text = tostring(count)
		end
	end
end)

-- Update state from gameplay
coins:Set(42)

A minimal lifecycle scope passed to each component function. Use Bind to subscribe to reactive state, BindProperty to drive a named instance property directly from state, BindEvent to connect instance events, Mount to compose child components, and MountWhen for conditional rendering. All bindings, event connections, and child mounts made through this scope are automatically cleaned up when the parent GUI is unmounted.

Component​

</>
type Component = (
instance: Instance,
scope: Scope,
properties: {[string]: any}?
) → ()

A reusable function that applies behavior to a GUI child Instance. Components are mounted to a specific Instance in the GUI hierarchy and can bind to reactive state using the provided Scope. Components should be used for reusable pieces of UI behavior. For example, you might have a "HealthBar" component that you mount to any Frame to turn it into a health bar, or a "Draggable" component that makes any GUI element draggable. Components should not manage the lifecycle of their own Instance (i.e. by destroying it or reparenting it), they should only be responsible for updating the instance's properties in response to state changes, passed down through properties or observed via Bind. Let the parent Controller manage the lifecycle of all instances.

State​

</>
interface State {}

A reactive state object. Create these using Flux:State and pass them to Controller:Bind or Scope:Bind inside components to reactively drive UI updates.

AnimatedState​

</>
interface AnimatedState {}

A read-only animated state returned by Flux:Tween and Flux:Spring. Supports Get, Observe, and Destroy — compatible with Bind and BindProperty. Call Destroy when you are finished with it to stop the animation and free resources.

Computed​

</>
Flux.Computed(
_self: Flux,
callback: () → T,--

A function that derives the computed value from current dependencies

dependencies: {State<any>}?--

Optional array of states to observe; if omitted, the callback is evaluated once

) → State<T>

Creates a reactive derived state value. The callback runs immediately to seed the value, then re-runs whenever any dependency changes. The returned state is read-only from the caller perspective and is cleaned up automatically when its dependencies are destroyed.

Compose​

</>
Flux.Compose(
_self: Flux,
components: {Component}--

Components to run in sequence for the same instance and scope

) → Component

Composes multiple components into one component. The returned component invokes each component in order using the same instance, scope, and properties. 

local FancyButton = Flux:Compose({
	HoverEffect,
	ClickEffect,
	PulseEffect,
})
controller:Mount(FancyButton, button, {theme = themeState})

Tween​

</>
Flux.Tween(
_self: Flux,
source: State<T>,--

The reactive state to animate

tweenInfo: TweenInfo--

Controls duration, easing style, and easing direction

) → AnimatedState<T>

Returns an AnimatedState that smoothly interpolates toward source whenever it changes, using TweenInfo for timing and easing. Supports number and any Roblox type with a :Lerp method (UDim2, Vector2, Vector3, Color3, CFrame, UDim). Call Destroy when finished. 

local health = Flux:State(100)
local animHealth = Flux:Tween(health, TweenInfo.new(0.4, Enum.EasingStyle.Quad))
scope:BindProperty(animHealth, bar, "Size")
health:Set(50)

Spring​

</>
Flux.Spring(
_self: Flux,
source: State<T>,--

The reactive state to animate

speed: number,--

Angular frequency — how quickly the spring responds (e.g. 15–25 for snappy UI)

damping: number--

Damping ratio — 1 is critically damped (no overshoot), < 1 is bouncy

) → AnimatedState<T>

Returns an AnimatedState driven by spring physics that follows source whenever it changes. Supports number, Vector2, Vector3, Color3, and UDim2. The Heartbeat connection is only active while the spring is moving and disconnects automatically when settled. Call Destroy when finished. 

local open = Flux:State(UDim2.fromScale(0, 0))
local animOpen = Flux:Spring(open, 20, 1)
scope:BindProperty(animOpen, panel, "Size")
open:Set(UDim2.fromScale(1, 1))
