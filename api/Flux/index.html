Flux

Client module for mounting and managing GUI instances. Each mount parents the GUI to PlayerGui and returns a Controller that owns the instance's lifecycle. The caller is responsible for cloning templates before passing them in. 

local Flux = require(ReplicatedStorage.Flux)

-- Create reactive state
local coins: Flux.State<number> = Flux:State(0)

-- Clone and mount the GUI
local gui = ReplicatedStorage.Assets.Gui.CoinScreenGui:Clone()
local controller = Flux:MountScreenGui(gui)

-- Bind state to UI
controller:Bind(coins, function(count: number)
	local frame = controller:GetInstance():FindFirstChild("Frame")
	if frame then
		local coinsLabel = frame:FindFirstChild("CoinsLabel")
		if coinsLabel and coinsLabel:IsA("TextLabel") then
			coinsLabel.Text = tostring(count)
		end
	end
end)

-- Update state from gameplay
coins:Set(42)

A minimal lifecycle scope passed to each component function. Use Bind to subscribe to reactive state and Mount to compose child components. All bindings and child mounts made through this scope are automatically cleaned up when the parent GUI is unmounted.

Component​

</>
type Component = (
instance: Instance,
scope: Scope,
props: {[string]: any}?
) → ()

A reusable function that applies behavior to a GUI child Instance. Components are mounted to a specific Instance in the GUI hierarchy and can bind to reactive state using the provided Scope. Components should be used for reusable pieces of UI behavior. For example, you might have a "HealthBar" component that you mount to any Frame to turn it into a health bar, or a "Draggable" component that makes any GUI element draggable. Components should not manage the lifecycle of their own Instance (i.e. by destroying it or reparenting it), they should only be responsible for updating the instance's properties in response to state changes, passed down through props or observed via Bind. Let the parent Controller manage the lifecycle of all instances.

A reactive state object. Create these using Flux:State and pass them to Controller:Bind or Scope:Bind inside components to reactively drive UI updates.
