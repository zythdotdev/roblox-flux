Controller

Controller for a mounted GUI. Returned by every Mount method call. A Controller manages the lifecycle of the GUI Instance and all reactive connections. Call Unmount to clean up, Bind to react to state changes, and Mount to apply reusable components that are automatically cleaned up when the controller is unmounted. Access the GUI hierarchy via GetInstance().

A lifecycle scope passed to each component function. Use Bind to subscribe to reactive state and Mount to compose child components. All bindings and child mounts made through this scope are automatically cleaned up when the parent GUI is unmounted.

Component​

</>
type Component = (
instance: Instance,
scope: Scope,
props: {[string]: any}?
) → ()

A reusable function that applies behavior to a GUI child Instance. Components are mounted to a specific Instance in the GUI hierarchy and can bind to reactive state using the provided Scope. Components should be used for reusable pieces of UI behavior. For example, you might have a "HealthBar" component that you mount to any Frame to turn it into a health bar, or a "Draggable" component that makes a GUI element draggable. Components should not manage the lifecycle of their own Instance (i.e. by destroying it or reparenting it), they should only be responsible for updating the instance's properties in response to state changes, passed down through props or observed via Bind. Let the parent Controller manage the lifecycle of all instances.

Unmount​

</>
Controller.Unmount(self: Self) → ()

Destroys the GUI Instance and disconnects all bindings. This should be called when you are finished with the GUI to properly clean up resources and prevent memory leaks. 

local controller = Flux:MountScreenGui(gui)
controller:Unmount() -- Clean up when done

Mount​

</>
Controller.Mount(
self: Self,
component: Component,--

The component function to apply

instance: Instance,--

The GUI instance to pass to the component

props: {[string]: any}?--

Optional props table to pass to the component

) → ()

Applies a reusable Component function to a child instance. The component receives a Scope that allows it to bind to reactive state. All bindings made by the component are automatically cleaned up when the controller is unmounted. 

local function HealthBar(instance: Instance, scope: Flux.Scope, props: {health: Flux.State<number>})
	scope:Bind(props.health, function(hp: number)
		local bar: Frame = instance :: Frame
		bar.Size = UDim2.fromScale(hp / 100, 1)
	end)
end

local healthState = Flux:State(100)
local healthBarFrame = controller:GetInstance():FindFirstChild("HealthBarFrame")
if healthBarFrame then
	controller:Mount(HealthBar, healthBarFrame, {health = healthState})
end

Bind​

</>
Controller.Bind(
self: Self,
state: State<T>,--

The reactive state to observe

callback: (newValue: T?) → ()--

Function to call with the current and future values

) → ()

Binds to a State object so that callback fires immediately with the current value, and then again every time the state changes. The binding is automatically cleaned up when the controller is unmounted. 

local coins: Flux.State<number> = Flux:State(0)
controller:Bind(coins, function(count: number)
	local label = controller:GetInstance():FindFirstChild("CoinsLabel")
	if label and label:IsA("TextLabel") then
		label.Text = tostring(count)
	end
end)
